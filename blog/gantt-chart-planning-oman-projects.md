## Dependency Types

### Finish-to-Start

Most common:

- Task B starts when Task A finishes
- Sequential work
- Clear handoff

### Start-to-Start

Parallel initiation:

- Task B starts when Task A starts
- Concurrent work
- Slight offset

### Finish-to-Finish

Parallel completion:

- Task B finishes when Task A finishes
- Coordinated endings
- Quality gates

### Start-to-Finish

Rare but useful:

- Task B can't finish until Task A starts
- Just-in-time scenarios

## Critical Path

The longest sequence:

### Identification

Finding critical path:

- Connected dependencies
- Longest duration
- Zero float
- Project end driver

### Significance

Why it matters:

- Delay here = project delay

- Resource priority
- Risk management

### Management

Using critical path:

- Monitor closely
- Protect from delays
- Accelerate if needed
- Communicate importance
